# 2.kubernetes

# 一 概述
首先有一个概念Orchestration(直译是管弦乐编曲，可以理解为协调器、编排)，随着容器技术的发展，我们需要一个东西来管理、扩展和维护容器应用，这个东西就是Orchestration。常见的有Kubernetes和Docker Swarm

## 1 简介
容器集群管理/容器调度/容器编排系统

# 三 基础
自动缩放功能，一旦服务需求减少，Kubernetes 就会自动减少每个服务的运行实例数量，而大多数容器只需要执行一个函数，当该函数执行完成之后就会停止。
